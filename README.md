## ü§ù Contribution Guidelines

1. **Branching**  
   - Create a new branch with a descriptive name (e.g., `feature-add-analysis`, `update-interview-data`).  

2. **Commit Messages**  
   - Write clear commit messages (e.g., `Added survey dataset`, `Updated interview transcript`).  

3. **Pull Requests**  
   - Submit a pull request for review before merging into `main`.  
   - Reviewer will check for conflicts and approve changes.  

4. **File Naming**  
   - Follow consistent naming conventions:  
     - Use lowercase with underscores (`survey_qs.docx`, `final_paper_v9999.docx`).  
     - Include version numbers or dates if multiple drafts exist.
